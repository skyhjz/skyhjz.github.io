设计模式-适配者模式

设计模式之适配器模式

1. 概述

适配器模式是一种结构型设计模式，其核心作用是将一个类的接口转换成客户端所期望的另一种接口。换句话说，适配器模式弥合了接口不兼容的类之间的鸿沟，使得原本由于接口不匹配而无法协同工作的类能够有效地协同工作。

2. 模式意图

  • 转换接口： 将一个类的接口转换成客户端需要的接口。
  • 兼容性： 使原本由于接口不兼容而不能一起工作的类可以一起工作。
  • 复用性： 复用现有的类，避免修改源代码。

3. 模式结构

适配器模式主要包含以下角色：

  • 目标接口（Target）： 定义客户端期望使用的接口。
  • 适配者（Adaptee）： 现有的类，其接口需要被适配。
  • 适配器（Adapter）： 适配器类，它实现了目标接口，并持有一个适配者类的实例，从而将适配者类的接口转换成目标接口。
  • 客户端（Client）： 通过目标接口使用适配器。

4. 适配器模式的两种实现方式

在 Java 中，适配器模式通常分为两种类型：类适配器和对象适配器。

4.1 类适配器

  • 实现方式： 类适配器通过多重继承实现。适配器类同时继承目标类并实现适配者接口。
  • 工作原理：
    • 通过继承目标类，适配器类可以访问目标类的方法和属性。
    • 通过实现适配者接口，适配器类可以将适配者的方法转换为目标类的方法。
  • 优缺点：
    • 优点： 简单直接。
    • 缺点： 只能适配类，不能适配对象；Java 不支持多重类继承，因此存在局限性。

4.2 对象适配器

  • 实现方式： 对象适配器通过组合实现。适配器类持有一个适配者类的实例，并同时实现目标接口。
  • 工作原理：
    • 通过持有适配者类的实例，适配器类可以调用适配者类的方法。
    • 通过实现目标接口，适配器类可以将适配者的方法转换为目标类的方法。
  • 优缺点：
    • 优点： 可以适配对象；可以使用接口或抽象类作为目标接口，灵活性更高。
    • 缺点： 相比类适配器，实现稍复杂。

5. 适用场景

  • 你想使用一个已经存在的类，但是它的接口不符合你的需求。
  • 你想创建一个可以复用的类，该类可以与不相关的类或不可预见的类协同工作。
  • 需要统一多个类的接口，将其转换为统一的接口。

6. 模式优点

  • 提高类的复用性： 适配器模式允许你使用现有的类，即使它们的接口不兼容。
  • 提高灵活性： 你可以根据需要创建多个适配器，以满足不同的接口需求。
  • 符合开闭原则： 你可以在不修改现有代码的情况下引入新的适配器。

7. 模式缺点

  • 增加代码复杂性： 适配器模式可能会增加代码的复杂性，特别是当适配器链很长时。
  • 过度使用： 过度使用适配器模式可能会导致系统难以理解和维护。

9. 总结

适配器模式是一种非常有用的设计模式，它可以帮助你解决接口不兼容的问题，并提高代码的复用性和灵活性。但是在使用过程中也需要注意适配器模式的缺点，避免过度使用。
